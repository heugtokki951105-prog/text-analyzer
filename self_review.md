# 자기 평가서

## 가장 잘했다고 생각하는 부분

이번 프로젝트에서 가장 잘한 부분은 텍스트 분석 기능을 `Document` 부모 클래스와 `SocialMediaPost` 자식 클래스로 나누어 설계한 점입니다. 공통 텍스트 처리 로직은 부모 클래스에 두고, 소셜 미디어에 특화된 해시태그와 멘션 추출 기능은 자식 클래스에 배치하여 역할을 분리했습니다.

또한 `_tokenize()` 비공개 메서드를 만들어 단어 분리 로직을 재사용하도록 했습니다. 이를 통해 `word_frequency()`와 `sentiment_score()`에서 같은 전처리 코드를 반복하지 않고 DRY 원칙을 적용할 수 있었습니다.

## 아쉬운 부분

현재 감정 분석 기능은 단순한 단어 사전 기반 방식이기 때문에 문맥을 고려하지 못합니다. 예를 들어 부정 표현이나 문장 전체의 의미를 분석하지 않고, 단어가 긍정 사전 또는 부정 사전에 있는지만 확인합니다.

또한 해시태그와 멘션 추출 기능은 공백 기준으로만 텍스트를 나누기 때문에 문장부호가 붙은 경우에는 결과가 완벽하지 않을 수 있습니다. 추후에는 정규표현식을 사용해 더 정확하게 추출하도록 개선하고 싶습니다.

## 다음에 개선하고 싶은 점

다음에는 CSV 또는 JSON 파일에서 텍스트 데이터를 불러와 여러 문서를 한 번에 분석하는 기능을 추가하고 싶습니다. 분석 결과를 표 형태로 저장하거나 시각화하면 실제 사용성이 더 좋아질 것 같습니다.

또한 테스트 커버리지를 더 넓혀 잘못된 타입, 특수문자, 긴 문장, 다양한 언어 입력에 대한 처리를 검증하고 싶습니다. GitHub Actions 같은 CI 도구를 연결하여 커밋할 때마다 자동으로 테스트가 실행되도록 만드는 것도 좋은 개선 방향입니다.

## AI 도구 사용 기록

프로젝트 요구사항 점검과 문서 및 테스트 작성 과정에서 AI 도구의 도움을 받았습니다. AI가 제안한 내용을 바탕으로 프로젝트 구조, 테스트 케이스, README 항목을 확인했고, 최종 내용은 과제 요구사항에 맞게 직접 검토했습니다.
